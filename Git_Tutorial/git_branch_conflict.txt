<git 브랜치 충돌(conflict)처리하기>

-> 충돌은 일반적으로 두개 이상의 브랜치에서 동일한 파일을 수정했을때, 그 내용이 서로 다를때 병합시 충돌이 발생할 수 있다.

-> 기본적으로 병합을 실행 할때는 master 브랜치에서 실시한다. / 만약 충돌 발생시 git merge develop을 실행하면 'merge conflict in ''이라는 경고문이 나온다.

-> 이때 해당 파일을 열어보면 git에서 자동으로 소스코드가 다른부분을 표시해준다. (HEAD = master / 하단 = 다른 브랜치)

-> 다른부분을 확인하고 어느 부분을 수정할 지 결정 후 다시 merge를 실행한다.

-> 항상 merge를 수행했다면 git branch -d 명령을 이용해 다른 브랜치를 지워준다. (삭제후 git branch로 확인하기).
